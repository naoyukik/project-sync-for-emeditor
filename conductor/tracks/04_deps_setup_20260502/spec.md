# Specification: プラグイン作成に必要なライブラリの調査と追加 (Issue #4)

## Overview
EmEditorのプロジェクトプラグインと連携するバックグラウンド監視プラグインを実装するため、必要な基盤ライブラリ（Rustクレート）を調査・選定し、プロジェクトに導入する。状態管理やエラーハンドリング等の追加クレートは現時点では見送り、必要最小限の構成とする。

## Functional Requirements
- **EmEditor SDK / Win32 API 連携**: `windows-rs` を用いて、EmEditorプラグインとして動作するための基盤（`DllMain`等）を構築できること。
- **ファイルシステム監視**: `notify` クレートを利用して、指定フォルダの変更を検知できる状態にすること。
- **依存関係の解決**: 選定したライブラリが `Cargo.toml` に追加され、競合なくビルドできること。

## Non-Functional Requirements
- **軽量性**: バックグラウンドで常駐するため、不要に重いフレームワークは避け、最小限の依存関係にとどめる。

## Acceptance Criteria
- [ ] 必要なライブラリ（`windows-rs`, `notify`等）が `Cargo.toml` に定義されている。
- [ ] Win32 FFI の設定が正しいことを確認するため、ダミーのエントリポイント（`DllMain`等）が実装されている。
- [ ] 上記を含めた状態で `cargo build` が成功する。

## Out of Scope
- 具体的なプラグインのビジネスロジック（自動登録処理など）の実装。
- ファイル監視の具体的なハンドリング処理。
